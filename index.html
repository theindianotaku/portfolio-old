My Workflow

Each project is unique so I follow an adaptive and iterative design process that suits the project’s requirement whether it be lean, agile or ordered. Each process varies depending upon the services required which includes web/mobile app design, user experience design and frontend development. Good design and code takes time so depending on the requirements, I can either start with the design with a pencil and notebook or start designing straight in the browser.

Having workflow or a design process provides me with a framework on how to best proceed when faced with a problem breaking it down into a series of interconnected systems. It is difficult to just highlight all the details in a single mockup, for example it can consist of many components, states, hidden elements, unseen feedback, user cases and scenarios, transitions and visual hints. To understand all of this, going through a design process provides me with a holistic overview.

My workflow can be broken down into 7 stages. There are many factors such as time constraints, pivoting and limited resources that can affect the outcome of each stage. The stages are so ordered that it includes a feedback-and-approval method before proceeding forward. This to me is a very crucial step in each stage which provides me with the understanding of the client towards my work. Below you can find an overview of how I typically approach a design plus code project. The code part is the most diverse step to give an overview on as it actually depends on the technical stack.

speaker_notes User Needs
  1. Study the business goals and the history of the client’s brand or organisation.
  2. Communicate with the client about the needs of the user.
  3. Communicate with the other team members to understand the technical resources available.
  4. Draft a scope to determine the checkpoints that align with the goals and fit within the budget, deadlines and technical resources.
speaker_notes Preliminary Analysis
  1. Explore the needs of the user
  2. Research the current competitors in the market
  3. Gather out points that might help to carry out the project
  4. Sketch out rough ideas.
speaker_notes UX Design
  1. Split down the views into smaller, actionable goals
  2. Research the existing UX patterns for achieving the similar goals/actions
  3. Understand the edge cases by building a persona and intent around each user type.
  4. Create user flows to determine how to complete each goal in the better possible way
  5. Break down the user flows into smaller interconnected components
  6. Develop wireframes based on the components
  7. Determine various states for each wireframe e.g. empty, filled, logged in, logged out, error etc.
speaker_notes Visual Design
  1. Research various design patterns and take inspirations
  2. Communicate with the client about the needs of the user
  3. Liaise with the engineers to understand the technical resources available
  4. Draft a scope to determine milestones that align with the goals and fit within the budget, deadline and technical resources
