Description

Helpdesk is the go-to app if you are confused or have any queries related to any app in Channel i. It comprises of well-documented FAQs and currently facing/unresolved issues related to any of the up-and-running services of Channel i.

My role in this project was to take up the already made wireframe and convert it into a developed UI. With the base user flow decided by the wireframes, I moved onto code the project.

I have used the base stylesheet of Channel i. The front-end was fairly simple. The application being a forum-type, there wasnâ€™t anything new to deploy in HTML/CSS part. The backend was developed in Django. The major portion of my time was spent in the integration of the backend and frontend. The working of Django Templates being a fairly new topic to me, took time to understand and implement. The app used AJAX to populate dynamic content. Not using any frontend templating frameworks puts a limitation on the code handling when AJAX content loading and DOM manipulation was considered. These were the only pullbacks in the project.

The project also included developing an Admin App which would enable the easy content creation and editing for the web app. Channel i being managed by IMG, only the members of IMG had the rights to add content to the Helpdesk. Therefore a proper management of the content was necessary to swiftly edit/add content. The basic aim of the Admin App was to add/edit articles like FAQs, existing issues and tutorials. The Admin App had a generic Material Design based UI. I have used Materialize CSS Framework in this. The framework was quite easy to understand and implement. And at that time, this framework had the best features when compared to the then-developing Material Design Lite CSS framework. Actually I did start the project using MDL, but dropped it midway.

CKEditor was used to add content as it was easily compatible with our existing development stack (Django) of Channel i. CKEditor was configured to use the Django File Manager (a simple plugin developed indigenously by IMG) so that the media management was easier as per the existing system.

There was a need to handle large/wide images in the application because, as per the base design guidelines the center-wrapper had a very low width profile. This made viewing images with smaller text difficult. So the best possible solution was to use a modal-view which would enable a larger size view. Throughout the Channel i, the Fancybox plugin was used. To use this plugin on the img elements, they had to be wrapped with an anchor a tag. As the content was marked down by the CKEditor plugin, to be able wrap all the img tags in the forum answers was a difficult task. Best possible solution to this was to employ a method in the CKEditor Config to markdown all the images in the content to be wrapped by a tags. This method had to replace the default. After some research and understanding from the CKEditor Documentation, it was implemented. This enabled all the images and videos in the content to be Fancybox plugable and enable a modal-view for larger size view.
